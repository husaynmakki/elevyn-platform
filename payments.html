Sending a document logs it here as "Sent." Actually collecting a real e-signature requires an e-signature provider (like the one built into GoldCore, or DocuSign/HelloSign) connected on a backend.

Connection Status

Pulls real completed payments from your backend and marks the matching invoice, POS sale, or payment link as paid here. Runs automatically when you open Billing, Point of Sale, or this page — use this button to check right now.

Backend Checkout Endpoint (recommended — supports any amount)

This should point at the deployed elevyn-payment-server (see the README in your download) — a small real backend that creates genuine Stripe Checkout Sessions for any dollar amount. Deploy it once, paste its URL here, and every "Collect Payment" / "Checkout" button across the whole platform will redirect to a real, live Stripe payment page.

Static Payment Link (fallback — fixed price only)

Create this in your Stripe Dashboard under Payment Links (or Square/PayPal's equivalent) — no code required. This is used automatically when no backend endpoint is configured above. Because it's a fixed-price link, it works well for things like a Day Pass or Gift Card purchase, but can't charge a specific invoice amount — the backend option above is what enables that.
